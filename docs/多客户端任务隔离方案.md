# 多客户端任务隔离方案

## 目标

新增 `GET /{client_id}/tasks` 端点，一次请求返回该客户端所有任务状态。不破坏现有 `GET /tasks/{task_id}/status`。

## 涉及文件

| 文件 | 改动 |
|------|------|
| `src/core/task_manager.py` | `create_task` 加 `client_id`；新增 `get_client_tasks(client_id)` |
| `src/api/routes/tasks.py` | `TaskSubmitRequest` 加 `client_id`；新增 `GET /{client_id}/tasks` |
| `src/api/routes/llm.py` | `GenerateRequest` 加 `client_id`，透传 |

## 不改的文件

| 文件 | 原因 |
|------|------|
| `scheduler.py` | 调度器不需要知道 client_id，只管 FIFO |
| `ws_proxy.py` / `ws_manager.py` | WS 推送到前端，与 client_id 无关 |
| `executors/*.py` | executor 不需要知道谁提交的 |

## TaskManager 改动

```python
# create_task 加参数
def create_task(cls, task_type: str, client_id: str = "") -> str:
    task_id = str(uuid.uuid4())
    with cls._lock:
        cls._tasks[task_id] = {
            "status": cls.STATUS_PENDING,
            "task_type": task_type,
            "client_id": client_id,
            "result": {},
            "created_at": time.time(),
        }
    return task_id

# 新增方法
@classmethod
def get_client_tasks(cls, client_id: str) -> list:
    """返回某个 client 的所有任务，按创建时间排序"""
    with cls._lock:
        tasks = [
            {"task_id": tid, "status": t["status"], "task_type": t["task_type"],
             "result": t["result"], "created_at": t["created_at"]}
            for tid, t in cls._tasks.items()
            if t.get("client_id") == client_id
        ]
        tasks.sort(key=lambda t: t["created_at"])
        return tasks
```

### 3. 新增端点

```python
@router.get("/{client_id}/tasks")
async def get_client_tasks(client_id: str):
    tasks = TaskManager.get_client_tasks(client_id)
    if not tasks:
        return error(f"No tasks found for client '{client_id}'", 404)
    return success(data={"tasks": tasks})
```

## 前端使用

```javascript
const CLIENT_ID = localStorage.getItem("echo_client")
                || crypto.randomUUID();

// 提交
await fetch("/interface/tasks/submit", {
  body: JSON.stringify({ task_type: "comfyui", client_id: CLIENT_ID, payload: {...} })
});

// 轮询（一个请求拿到全部任务）
setInterval(async () => {
  const { data } = await fetch(`/interface/${CLIENT_ID}/tasks`).then(r => r.json());
  data.tasks.forEach(t => updateTaskUI(t.task_id, t.status, t.result));
}, 2000);

// WS 照常用，不受影响
const ws = new WebSocket(`ws://host/interface/tasks/${taskId}/ws`);
```

## WS 与轮询的关系

- WS：executor 实时推送 `executing`/`progress`/`task_complete`，用于 comfyui 实时进度
- `GET /{client_id}/tasks`：2 秒轮询，拿到全部任务快照，用于 UI 状态更新
- 两者同时用不冲突——WS 管实时，轮询管全局状态

## 验证

1. 跑 `test/` 下的测试脚本，确认 client_id 透传
2. 两个不同的 client_id 提交任务，各自查 `/{client_id}/tasks`，互不看到对方任务
3. 不传 client_id 时，`GET //tasks` 返回空（或所有无 client_id 的任务）
4. WS 和轮询同时使用，TaskManager 线程安全无问题
